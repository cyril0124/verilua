# Install Verilua
本节主要介绍如何安装 Verilua。

Verilua 支持两种安装方式：
1. **Release 安装**（推荐）：使用预编译的 Release 包，无需编译，快速安装
2. **源码安装**：从源码编译安装，适合需要自定义或开发的场景

## 方式一：使用 Release 安装（推荐）

这是最简单的安装方式，直接下载预编译的 Release 包即可使用。

:::tip[获取最新构建包]
除了下载正式发布的 Release 包外，你也可以从 GitHub Actions 的 artifacts 中获取最新的构建包。每当有新的 commit 推送到主分支时，CI 会自动运行回归测试并生成完整的 release 包。由于正式 Release 包的发布通常不及时，从 artifacts 获取可以让你使用最新的功能和修复。

访问 [Actions 页面](https://github.com/cyril0124/verilua/actions/workflows/regression.yml)，选择最新的成功运行的 "Regression" 工作流，下载以下 artifact 之一：
- `verilua-x64-ubuntu-22.04-xxxxxxx` (Ubuntu 22.04)
- `verilua-x64-centos-7-xxxxxxx` (CentOS 7)

其中 `xxxxxxx` 是 commit 的短哈希。这些包包含完整的预编译 Verilua 环境，可以直接使用。

如果没有找到适合你系统的包，可以使用 CentOS 7 的包，因为 CentOS 7 使用较低版本的 glibc 2.17，通常能够兼容更高版本 glibc 的系统。
:::

### 1. 下载并解压

```bash
# 下载最新的 Release 包（以 Ubuntu 22.04 为例）
wget https://github.com/cyril0124/verilua/releases/download/v2.0.0/verilua-x64-ubuntu-22.04.zip

# 解压
unzip verilua-x64-ubuntu-22.04.zip
```

:::tip[其他平台]
请前往 [GitHub Releases](https://github.com/cyril0124/verilua/releases) 页面查看所有可用的预编译包。
:::

### 2. 激活 Verilua 环境

```bash
# 进入解压后的目录
cd verilua-x64-ubuntu-22.04

# 激活 Verilua 环境
source verilua.sh
```

### 3. 验证安装

```bash
# 运行测试命令验证安装是否成功
test_verilua
```

如果 Verilua 安装成功，则应该会看到 Terminal 最下面有类似的输出：
```shell title="Terminal"
Test Results:
  ✓ Passed: 2
  ✗ Failed: 0

✓  All verilua tests passed!
```

## 方式二：从源码安装

如果你需要自定义编译或参与开发，可以选择从源码安装。为了确保安装能够成功，需要全程确保网络环境可用。

### Prerequisites
- gcc >= 11.0（需要支持 C++20）
- 安装 [xmake](https://xmake.io/)
- 安装 [rust](https://www.rust-lang.org/tools/install)
- 安装 python3.xx
- 根据需求安装下列硬件仿真器
    - [verilator](https://github.com/verilator/verilator) >= 5.026
    - [iverilog](https://github.com/steveicarus/iverilog)

        :::danger[iverilog 版本问题]
        由于 iverilog 在 [v12_0 release 版本](https://github.com/steveicarus/iverilog/releases/tag/v12_0) 下存在 VPI 的 [cbNextSimTime 无限循环问题](https://github.com/steveicarus/iverilog/pull/1098)，会对 Verilua 的多任务调度功能造成影响。所以需要使用目前 iverilog 官方仓库下的 master 分支版本（或者叫 v13-devel 版本, 经过测试 `s20250103` 这个 tag 也可以用），这需要自行编译安装！

        安装完成后，需要设置环境变量 `IVERILOG_HOME` 指向刚刚安装的 iverilog 目录（包含 `bin` 和 `lib` 的目录）。

        :::note[iverilog 安装命令]
        ```shell
            git clone https://github.com/steveicarus/iverilog.git
            cd iverilog
            git checkout s20250103
            sh autoconf.sh
            ./configure
            make
            sudo make install
        ```
        :::

    - Synopsys VCS
    - Cadence Xcelium

### 安装步骤
```shell
xmake install verilua
```
:::warning[安装失败]
如果安装过程失败了，那么可以重新执行上述命令，尝试再次安装。
:::

### 测试安装是否成功
使用 shell 打印出 `VERILUA_HOME` 这一环境变量，如果 `VERILUA_HOME` 指向了当前的 Verilua 工程目录，那么就说明安装成功了。
也可以执行下面命令进行更完整的安装测试：
```shell
xmake run test
```
如果 Verilua 安装成功，则应该会看到 Terminal 最下面有这样的输出：
```shell title="Terminal"
  _____         _____ _____ 
 |  __ \ /\    / ____/ ____|

 |  ___/ /\ \  \___ \\___ \ 
 | |  / ____ \ ____) |___) |

```
